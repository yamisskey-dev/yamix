# YAMI トークン経済システム（Validation Economy）

## 概要

**YAMI トークン経済システム** は、心理的承認（Validation）を交換媒体とする経済システムである。

相談者は承認を求め、回答者は承認を与える。「あなたの気持ちは valid（妥当）である」という承認の交換を、YAMIトークンを介して可視化・制御する。

### Validation のダブルミーニング

「Validation（承認）」には二重の意味がある：

| 文脈 | 意味 |
|------|------|
| 心理学 | 承認・肯定（Emotional Validation） |
| IT | 妥当性検証（Data Validation） |

この二重性は意図的であり、「妥当性を認める」という本質を共有している。YAMIはこの「承認」を可視化したトークンである。

## 理論的根拠

### 1. ゲゼルの減価貨幣（Demurrage Currency）

シルビオ・ゲゼル（1862-1930）「自然的経済秩序」より：

> 「貨幣は交換の媒体であり、価値の保存手段ではない」

- 貨幣の**保蔵（hoarding）**が経済停滞の原因
- 保有コスト（demurrage）で**流通を強制**
- 実例：ヴェルグルの奇跡（1932年）、WIR銀行（1934年〜）

**承認経済への適用**：Validationは時間経過で減衰し、蓄積を抑制する。

### 2. メカニズムデザイン（2007年ノーベル経済学賞）

ハーヴィッツ、マスキン、マイヤーソンの理論：

- **インセンティブ整合性**：望ましい行動が個人の最適戦略になる設計
- **個人合理性**：参加することが各人にとって利益になる

**承認経済への適用**：回答することが報酬につながり、協力が合理的になる。

### 3. 行動経済学（2002年, 2017年ノーベル経済学賞）

**プロスペクト理論（カーネマン&トヴェルスキー）**：
- **損失回避**：損失は同額の利得の約2倍痛い
- 減衰 = 損失 → 「使わないと損」という強い動機

**ナッジ理論（セイラー）**：
- デフォルト設計で望ましい行動を促す
- 減衰は「使う」方向へのナッジとして機能

### 4. ロールズの正義論（1971）

**マキシミン原則**：
> 「最も恵まれない人の状況を最大化すべき」

**承認経済への適用**：
- 最も恵まれない = 相談が必要な人
- BI（ベーシックインカム）で基本アクセスを保証
- 「病んでいる人が貧しくなる」設計を回避

### 5. ゲーム理論

**フリーライダー問題**：
- 報酬なし → 「回答せず相談だけ」が合理的（協力崩壊）
- 報酬あり → 回答が合理的になる

**承認経済への適用**：回答報酬でフリーライダーを抑制しつつ、減衰で格差を抑制。

## 設計原則

### 核心的ジレンマ

| 問題 | 解決策 |
|------|--------|
| 報酬なし → 善意の搾取 | 回答報酬を設ける |
| 報酬あり → 格差が生まれる | 減衰で蓄積を抑制 |
| 無制限依存 | 相談にコストを課す |
| 困窮者の排除 | BIで基本アクセスを保証 |

### 設計要件

1. **インセンティブ整合性**：回答が個人の最適戦略になる
2. **フリーライダー防止**：相談だけでは持続不可能
3. **善意の保護**：報酬が内発的動機を駆逐しない
4. **格差抑制**：蓄積による格差が生まれにくい
5. **基本アクセス保証**：困っている人が排除されない

## 経済パラメータ

### パラメータ定義

```typescript
interface ValidationEconomyParams {
  // ベーシックインカム（BI）
  dailyGrant: number;        // 毎日付与量

  // 減衰（Demurrage）
  decayRate: number;         // 日次減衰率 (0-1)

  // 相談コスト
  aiConsultCost: number;     // AI相談
  humanConsultCost: number;  // 人間相談

  // 報酬
  responseReward: number;    // 回答報酬
  dailyRewardCap: number;    // 1日の報酬上限

  // 制約
  initialBalance: number;    // 新規ユーザーの初期残高
  maxBalance: number;        // 残高上限
}
```

### 数学的関係

**均衡残高の計算**：

```
均衡残高 = dailyGrant / decayRate
```

何も行動しない場合、残高は均衡残高に収束する。

**設計制約**：

```
1. 均衡残高 > humanConsultCost × 最低保証回数
   （基本アクセスを保証）

2. dailyRewardCap < dailyGrant × 2
   （報酬が支配的にならない）

3. responseReward < humanConsultCost
   （相談コストと報酬のバランス）
```

### 推奨パラメータ（初期値）

| パラメータ | 値 | 根拠 |
|------------|-----|------|
| dailyGrant | 10 | 人間相談2回分/日 |
| decayRate | 0.2 (20%) | 均衡50、5日で67%減 |
| initialBalance | 50 | 均衡残高と同じ（100%スタート） |
| maxBalance | 100 | 均衡の2倍 |
| aiConsultCost | 1 | 最小単位 |
| humanConsultCost | 5 | AI×5（時間の価値） |
| responseReward | 3 | 人間相談の60% |
| dailyRewardCap | 15 | 5回分 |

**シミュレーション（何もしない場合）**：

```
Day 0:  50.0 V（初期 = 均衡残高）
Day 1:  50 × 0.8 + 10 = 50.0 V
Day 2:  50 × 0.8 + 10 = 50.0 V
...
収束:   50.0 V（= 10 / 0.2）

※ 初期残高を均衡残高と同じ50に設定することで、
   新規ユーザーが初回から100%の状態でスタートできる
```

## 格差抑制メカニズム

### なぜ格差が生まれにくいか

1. **減衰による平準化**：
   - 全員の残高が時間経過で減少
   - 蓄積してもすぐに減る
   - 長期的には均衡残高に収束

2. **BIによる底上げ**：
   - 全員に毎日一定量を付与
   - 残高ゼロにはならない

3. **報酬上限**：
   - 1日の報酬獲得に上限
   - 「稼ぎまくる」ことができない

4. **残高上限**：
   - 最大保有量を制限
   - 極端な富の蓄積を防止

### ゲーム理論的均衡

- 何もしない → 均衡残高で安定
- 回答する → 一時的に増えるが減衰で戻る
- 相談する → 減るがBIで回復
- → 誰も「極端に豊か/貧しく」ならない

## YAMI DAOとの関係

現在のYAMIトークンは **オフチェーン（アプリ内データベース）** で管理されています。

将来的にブロックチェーン対応する場合は、[YAMI DAO](https://github.com/yamisskey-dev/yamidao) と連携します：

| 項目 | YAMI（現在） | YAMI DAO連携（将来） |
|------|------------|------|
| 管理 | Yamix アプリ内 | YAMI DAO経由でOptimismチェーン |
| アカウント | Misskey連携 | YAMI DAOでETHアドレス管理を委譲 |
| 減衰 | あり | オフチェーン部分のみ適用 |
| BI | あり | オフチェーン部分のみ適用 |

**設計方針**：
- ETHアドレス管理はYAMI DAOに委譲し、Yamixではブロックチェーン機能を直接実装しない
- MisskeyアカウントとETHアドレスの連携はYAMI DAOが提供
- 承認経済の減衰・BIメカニズムはYamixアプリ内で継続

## よくある誤解と反論

### 誤解1: 「病んでいる人が貧しくなる」

**事実**：BIと減衰により、全員が均衡残高に収束する。相談が多くても、毎日の付与で回復する。

### 誤解2: 「回答者が富を蓄積する」

**事実**：減衰により蓄積は困難。報酬上限もあるため、「稼ぎまくる」ことはできない。

### 誤解3: 「善意が金銭化される」

**事実**：報酬は「感謝の印」であり、内発的動機を補完する。報酬額はコストより低く設定され、純粋な金銭目的では割に合わない。

### 誤解4: 「貧富の格差が生まれる」

**事実**：減衰+BI+上限により、全員が同じ水準に収束する設計。

### 誤解5: 「メンタルの弱さのマウント合戦が起きる」

**事実**：「より病んでいる」ことはトークン獲得に繋がらない。むしろ、より多く相談する（より多く承認を要求する）とトークンが減る。競争が起きるとすれば、「より良い回答をする」方向である。

### 誤解6: 「AI依存を助長している」

**事実**：AIへの依存は、人間への不健全な依存を代替するもの。人間に一方的に依存する構造は双方を消耗させるが、AIは消耗しないため「依存先」として機能できる。これにより人間同士の関係が健全化される。

## FAQ

### Q: 本当に辛いときにトークンがなかったらどうするの？

A: 毎日BIが付与されるため、AI相談10回分は常に保証される。また、危機検出機能により、深刻な状況では相談窓口が自動的に案内される。

### Q: 回答で稼いだトークンで無限に相談できない？

A: 数学的には可能だが、「回答できる健全さ」と「相談が必要な状態」は通常両立しにくいため、自然な抑制が働く。また、回答で得たトークンを相談で使うこと自体は、システムが意図した「相互扶助」の形である。

### Q: なぜAI相談は安いの？

A: AI相談のコストは純粋に運営費（API利用料等）のみ。人間の時間や精神的労働は発生しないため、コストを低く設定している。

### Q: トークン残高は公開される？

A: いいえ。トークン残高は非公開であり、「トークンを多く持っている = 偉い」という社会的地位化を防ぐ。

### Q: なぜ「YAMI」という名前？

A: YAMI（闇）は、精神的な暗闇を可視化し、共有し、支え合うという本プラットフォームの理念を表現しています。技術的には「Validation（承認）」を実装していますが、ユーザー向けには「YAMI」として統一して表示します。
