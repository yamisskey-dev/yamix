# YAMI トークン経済設計

## 概要

YAMIトークンは、Yamixにおける「精神的リソース」の交換を可視化・制御するための仕組みです。

**核心的な設計原理**: 他者の注意や時間を借りるにはコストがかかる。

---

## トークンの流れ

| 行動 | コスト/報酬 | 説明 |
|------|------------|------|
| AI相談 | **-1 YAMI** | 運営コストのみ |
| 人間相談 | **-5 YAMI** | 他者の時間と精神を借りる対価 |
| 人間の回答報酬 | **+4 YAMI** | 相談に応じた人への報酬 |
| 一般投稿 | 無料 | 自己表現にはコストなし |
| 自分への返信 | 無料 | 自分の投稿への返信は無料 |
| 毎日の無料付与 | **+3 YAMI** | 基本的な相談は無料でアクセス可能 |

### トークンの方向

```
┌─────────────────┐                    ┌─────────────────┐
│   相談する人     │  ─── YAMI消費 ───▶ │   システム       │
│ （注意を要求）   │                    │                 │
└─────────────────┘                    └────────┬────────┘
                                                │
                                                │ YAMI付与
                                                ▼
                                       ┌─────────────────┐
                                       │   回答する人     │
                                       │ （注意を提供）   │
                                       └─────────────────┘
```

**重要**: 相談する側がYAMIを**消費**し、回答する側がYAMIを**獲得**します。

## 設計思想

### 0. AIによる依存の代替

人間同士の関係における不健全な依存は、双方を消耗させます。

- 相談する側: 「迷惑をかけている」という罪悪感
- 相談される側: 精神的負担、善意の搾取

Yamixは、この問題をAIで代替することで解決を図ります。

| 従来の構造 | Yamixの構造 |
|-----------|------------|
| 人間 → 人間（一方的な依存） | 人間 → AI（負担なし） |
| 相談される側が消耗 | AIは消耗しない |
| 善意が搾取される | 人間の回答者には報酬 |

**AIは「依存先」として機能することで、人間同士の関係を健全化します。**

- AI相談は安価（1 YAMI）で、基本的な相談ニーズを満たす
- 人間相談は高価（5 YAMI）だが、回答者に報酬が支払われる
- 人間に一方的に依存する構造を、AIが吸収する

### 1. 依存の無制限化を防ぐ

相談にはコストがかかるため、無制限に他者の注意を要求することはできません。

- 毎日+3 YAMIの付与で、AI相談3回または人間相談0.6回分が保証される
- それ以上の相談は、過去に蓄積したYAMI、または購入したYAMIが必要
- 注意を要求すればするほど、要求できる量が減っていく

### 2. 回答者が報われる

精神的な労働には対価が支払われるべきです。

- 相談に応じた人は+4 YAMIを獲得
- 相談料5 YAMIのうち、4 YAMIが回答者へ、1 YAMIがシステム維持費
- 「善意の搾取」ではなく「持続可能な相互扶助」

### 3. 価値の可視化

他者の時間と精神には価値があります。

- 人間相談（5 YAMI）> AI相談（1 YAMI）
- この差が「人間の時間の価値」を表現している
- コストの存在が、相談の質を高める効果も期待できる

### 4. 基本アクセスの保証

本当に困っている人が排除されないよう、毎日の無料付与があります。

- 毎日+3 YAMIで、最低限の相談は常に可能
- Optimism ETHでの購入も可能（将来実装）

## よくある誤解と反論

### 誤解1: 「病んでいる人が注目を集めるほどトークンが得られる」

**事実は逆です。**

- 相談する側（注意を要求する側）はYAMIを**消費**します
- 病んでいる人が他者の注意を要求すればするほど、YAMIは**減っていきます**
- トークンを獲得するのは、相談に**応じる側**です

### 誤解2: 「メンタルの弱さのマウント合戦が起きる」

**設計上、起きません。**

- 「より病んでいる」ことはトークン獲得に繋がりません
- むしろ、より多く相談する（より多く注意を要求する）とトークンが減ります
- 競争が起きるとすれば、「より良い回答をする」方向です

### 誤解3: 「依存の奪い合いになる」

**設計上、逆の効果があります。**

- 依存的に他者の注意を要求するほど、コストがかかります
- 無制限に依存することは、システム上不可能です
- これは「依存を助長する」のではなく「依存にブレーキをかける」設計です

### 誤解4: 「トークン化 = 病みの通貨化」

**通貨化されるのは「他者の注意」です。**

- トークンが表現しているのは「他者の時間と精神的労働の価値」
- 病んでいることではなく、「他者の注意を借りること」にコストがかかる
- 自己表現（一般投稿）は無料であり、「病み」自体には課金されない

### 誤解5: 「AI依存を助長している」

**AIへの依存は、人間への不健全な依存を代替するものです。**

- 人間に一方的に依存する構造は、双方を消耗させる
- AIは消耗しないため、「依存先」として機能できる
- これにより人間同士の関係が健全化される

野良のAI（ChatGPT等に個人的にプロンプトを設定して依存する形態）と比較して、Yamixは：

| 観点 | 野良のAI依存 | Yamix |
|------|-------------|-------|
| 透明性 | プロンプト非公開 | プロンプト公開 |
| コスト認識 | 無制限に見える | コストが明示される |
| 依存抑制 | なし | トークン消費による自然な制限 |
| 人間の報酬 | なし | 回答者に報酬が支払われる |

## FAQ

### Q: 本当に辛いときにYAMIがなかったらどうするの？

A: 毎日+3 YAMIが付与されるため、AI相談3回分は常に保証されます。また、危機検出機能により、深刻な状況では相談窓口が自動的に案内されます。

### Q: 回答で稼いだYAMIで無限に相談できない？

A: 数学的には可能ですが、「回答できる健全さ」と「相談が必要な状態」は通常両立しにくいため、自然な抑制が働きます。また、回答で得たYAMIを相談で使うこと自体は、システムが意図した「相互扶助」の形です。

### Q: なぜAI相談は安いの？

A: AI相談のコストは純粋に運営費（API利用料等）のみです。人間の時間や精神的労働は発生しないため、コストを低く設定しています。

### Q: トークン残高は公開される？

A: いいえ。トークン残高は非公開であり、「トークンを多く持っている = 偉い」という社会的地位化を防ぎます。
